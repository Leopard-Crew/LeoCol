LRMErrorMake

Creates a basic LeoRM NSError. 

NSError *LRMErrorMake(
    NSInteger code,
    NSString *message);
Parameters
code
LeoRM error code.
message
Human-readable error message. If nil or empty, a default message is used.
Return Value

Returns an autoreleased NSError using LRMErrorDomain.

Discussion

Use this helper for LeoRM-level errors that are not direct SQLite failures, for example invalid arguments or invalid object lifecycle state.

LRMErrorMakeWithUserInfo

Creates a LeoRM NSError with additional userInfo entries. 

NSError *LRMErrorMakeWithUserInfo(
    NSInteger code,
    NSString *message,
    NSDictionary *extraUserInfo);
Parameters
code
LeoRM error code.
message
Human-readable error message. If nil or empty, a default message is used.
extraUserInfo
Optional dictionary merged into the returned error's userInfo.
Return Value

Returns an autoreleased NSError using LRMErrorDomain.

Discussion

Values in extraUserInfo are added after the localized description is set. Passing nil for extraUserInfo is valid.

LRMSQLiteErrorMake

Creates a LeoRM NSError from a SQLite failure. 

NSError *LRMSQLiteErrorMake(
    struct sqlite3 *database, 
    int sqliteCode, 
    NSString *sql, 
    NSString *databasePath);
Parameters
database
SQLite database handle. May be NULL, but when provided it is used to retrieve sqlite3_errmsg().
sqliteCode
Raw SQLite result code.
sql
Optional SQL string related to the failure.
databasePath
Optional database path related to the failure.
Return Value

Returns an autoreleased NSError using LRMErrorDomain and code LRMErrorSQLite.

Discussion

The returned error includes LRMErrorSQLiteCodeKey and LRMErrorSQLiteMessageKey. It includes LRMErrorSQLKey and LRMErrorDatabasePathKey only when those values are available.
