LRMRow

Foundation-shaped access to the current SQLite result row.

Superclass: NSObject
Declared In: LRMRow


LRMRow reads values from the current sqlite3_stmt row state. It does not own the SQLite statement and does not remain valid independently of the LRMResultSet that created it.
Rows are lightweight views over the current result row. The caller should read needed values before advancing or closing the result set.
LRMRow does not map values into domain objects. Domain mapping belongs in a repository or domain store above LeoRM.
