applyToDatabase:error:

Applies this migration's SQL statements to an open database. 

- (BOOL)applyToDatabase:(LRMDatabase *)database error:(NSError **)error;
Parameters
database
Open LRMDatabase. Must not be nil and must be open.
error
Optional NSError output. Filled on invalid migration data, invalid database state, prepare failure, or execution failure.
Return Value

Returns YES when all SQL statements execute successfully, NO on the first failure.

Discussion

This method does not start or commit a transaction. LRMMigrationRunner owns transaction boundaries for normal migration execution.

fromVersion

Returns the source schema version. 

- (NSInteger)fromVersion;
Return Value

Source version for this migration step.

- (id)initFromVersion:(NSInteger)fromVersion toVersion:(NSInteger)toVersion SQLStatements:(NSArray *)SQLStatements;
Parameters
fromVersion
Source schema version.
toVersion
Target schema version.
SQLStatements
Ordered array of SQL statements. The array is copied.
Return Value

Returns an owned LRMMigration.

Discussion

Validation of SQL statement contents happens when the migration is applied.
