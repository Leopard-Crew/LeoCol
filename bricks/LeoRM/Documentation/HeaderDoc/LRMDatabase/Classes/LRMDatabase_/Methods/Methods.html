beginTransaction:

Begins an explicit SQLite transaction. 

- (LRMTransaction *)beginTransaction:(NSError **)error;
Parameters
error
Optional NSError output. Filled when the database is closed or the transaction cannot be started.
Return Value

Returns an autoreleased active LRMTransaction, or nil on failure.

Discussion

Nested transactions are not part of LeoRM V1. Call commit: or rollback: on the returned transaction to end it.

close

Closes the SQLite database connection if open. 

- (void)close;
Discussion

This method is idempotent for already closed databases. If SQLite refuses to close because statements are still active, the internal handle is kept so that later cleanup can still happen. Callers should close result sets and finalize statements before closing the database.

databaseWithPath:error:

Creates an autoreleased database object for a path. 

+ (id)databaseWithPath:(NSString *)path error:(NSError **)error;
Parameters
path
SQLite database path. Must not be nil or empty. The special SQLite path ":memory:" is valid.
error
Optional NSError output. Filled when the path is invalid.
Return Value

Returns an autoreleased LRMDatabase, or nil on failure.

Discussion

This method does not open the SQLite connection. Call open: explicitly.

ensureMetadataTable:

Ensures that LeoRM's small metadata table exists. 

- (BOOL)ensureMetadataTable:(NSError **)error;
Parameters
error
Optional NSError output.
Return Value

Returns YES on success, NO on failure.

Discussion

The metadata table is named lrm_metadata and stores string key/value pairs. It is LeoRM-owned housekeeping data, not a private store format.

getSchemaVersion:forSchemaName:error:

Reads the stored schema version for a named schema. 

- (BOOL)getSchemaVersion:(NSInteger *)version forSchemaName:(NSString *)schemaName error:(NSError **)error;
Parameters
version
Output pointer receiving the schema version. Must not be NULL.
schemaName
Schema name. Must not be nil or empty.
error
Optional NSError output.
Return Value

Returns YES on success, NO on failure.

Discussion

Missing schema versions read as 0. This is used by LRMMigrationRunner to start migration from an empty database.

initWithPath:error:

Initializes a database object for a path. 

- (id)initWithPath:(NSString *)path error:(NSError **)error;
Parameters
path
SQLite database path. Must not be nil or empty. The special SQLite path ":memory:" is valid.
error
Optional NSError output. Filled when the path is invalid.
Return Value

Returns an initialized owned LRMDatabase, or nil on failure.

Discussion

This initializer copies the path but does not open the SQLite connection. Call open: explicitly.

isOpen

Reports whether the SQLite handle is currently open. 

- (BOOL)isOpen;
Return Value

Returns YES when the database has an active sqlite3 handle.

metadataValueForKey:error:

Reads a metadata value. 

- (NSString *)metadataValueForKey:(NSString *)key error:(NSError **)error;
Parameters
key
Metadata key. Must not be nil or empty.
error
Optional NSError output.
Return Value

Returns an autoreleased NSString value, or nil when the key is not present or when an error occurs.

Discussion

A missing key is not treated as an error. Because nil can mean either "missing" or "failure", callers that need to distinguish those cases should inspect the supplied NSError pointer.

open:

Opens the SQLite database connection. 

- (BOOL)open:(NSError **)error;
Parameters
error
Optional NSError output. Filled when sqlite3_open fails.
Return Value

Returns YES on success, NO on failure.

Discussion

Calling open: on an already open database returns YES. SQLite failures are reported through LRMSQLiteErrorMake. The database remains closed on failure.

path

Returns the SQLite database path. 

- (NSString *)path;
Return Value

Returns the path copied during initialization.

Discussion

The caller does not own the returned object.

prepareStatement:error:

Prepares SQL as an LRMStatement. 

- (LRMStatement *)prepareStatement:(NSString *)sql error:(NSError **)error;
Parameters
sql
SQL string to prepare. Must not be nil or empty.
error
Optional NSError output. Filled on invalid state or SQLite prepare failure.
Return Value

Returns an autoreleased LRMStatement, or nil on failure.

Discussion

The database must be open. SQL remains visible to the caller and is not generated or hidden by LeoRM.

setMetadataValue:forKey:error:

Writes or deletes a metadata value. 

- (BOOL)setMetadataValue:(NSString *)value forKey:(NSString *)key 
        error:(NSError **)error;
Parameters
value
Metadata value. Passing nil deletes the key.
key
Metadata key. Must not be nil or empty.
error
Optional NSError output.
Return Value

Returns YES on success, NO on failure.

Discussion

Values are stored as TEXT. Domain data should not be stored in LeoRM metadata.

setSchemaVersion:forSchemaName:error:

Stores the schema version for a named schema. 

- (BOOL)setSchemaVersion:(NSInteger)version forSchemaName:(NSString *)schemaName error:(NSError **)error;
Parameters
version
Schema version. Must not be negative.
schemaName
Schema name. Must not be nil or empty.
error
Optional NSError output.
Return Value

Returns YES on success, NO on failure.

Discussion

Migration code should update the schema version only after the corresponding migration step has succeeded.
